Project Overview

Understanding the problem, motivation, and research objective

Problem Context

Monocular depth estimation provides a low-cost alternative to hardware-based sensing systems such as LiDAR for assistive navigation. However, achieving real-time performance on edge-constrained devices remains a major challenge.

Research Objective

This project investigates whether model architecture, rather than input resolution, governs the trade-off between inference speed and structural consistency under deployment constraints.

Why This Matters

In assistive navigation, latency and structural reliability are safety-critical. A model that is fast but unreliable can be dangerous, while a model that is accurate but too slow may be unsuitable for real-world deployment.

Core Study Design

The research compares a transformer-based model (Depth Anything V2 Small) and a CNN-based model (MiDaS Small) using a controlled benchmark on RTX 2050, supported by cross-device validation on Apple M1 and Raspberry Pi systems.

Research Question

Does architecture drive the speed–accuracy trade-off more strongly than input resolution?

Models Compared

Depth Anything V2 Small vs MiDaS Small

Deployment Focus

Edge-constrained, safety-critical assistive navigation

Key Outcome

No evaluated configuration achieved both real-time speed and acceptable structural reliability.
